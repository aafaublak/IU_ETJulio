Clases para Validaciones

Validations Validations_Class.js

Clase que contiene todas las validaciones atomicas sobre atributos de formulario. Cada metodo recibe el ID del elemento HTML a validar (tal como pide el enunciado) y se encarga el mismo de leer el valor del DOM.

readValue(elementId)

Devuelve el valor textual del elemento HTML segun su tipo. Para inputs file devuelve el nombre del fichero seleccionado o cadena vacia; para checkbox devuelve "true"/""; para grupos radio devuelve el value del radio marcado.

ParametroTipoDescripcion
elementIdstringID del elemento HTML.

Retorna: string|null — cadena con el valor o null si el elemento no existe.

readFile(elementId)

Devuelve el primer File seleccionado en un input type=file. Lo utilizan internamente las validaciones de fichero (no_file, format_name_file, type_file, max_size_file) para no leer el DOM de varias formas distintas.

ParametroTipoDescripcion
elementIdstringID del input file.

Retorna: File|null — fichero seleccionado o null si no hay.

min_size(elementId, minSize)

Valida que el valor del elemento tenga al menos minSize caracteres. Si el elemento no existe en el DOM devuelve false (la validacion no se ha podido realizar).

ParametroTipoDescripcion
elementIdstringID del elemento HTML
minSizenumberTamano minimo requerido

Retorna: boolean — true si el valor tiene longitud >= minSize

max_size(elementId, maxSize)

Valida que el valor del elemento no exceda maxSize caracteres. Si el elemento no existe en el DOM devuelve false.

ParametroTipoDescripcion
elementIdstringID del elemento HTML
maxSizenumberTamano maximo permitido

Retorna: boolean — true si el valor tiene longitud <= maxSize

format(elementId, regexpStr)

Valida que el valor del elemento (leido con readValue) case con la expresion regular indicada. Si el elemento no existe en el DOM o la expresion regular no es una cadena no vacia, devuelve false porque la validacion no se ha podido aplicar.

ParametroTipoDescripcion
elementIdstringID del elemento HTML
regexpStrstringExpresion regular como cadena

Retorna: boolean — true si el valor cumple el formato.

date(elementId)

Valida que el valor del elemento sea una fecha real en formato dd/mm/aaaa. Comprueba dias por mes, anos bisiestos y limites de cada mes sin restriccion de rango de anos (por ejemplo 31/02/2024 y 30/04/1800 no son validas). Se usa en la estructura de project para los campos start_date_project y end_date_project en ADD y EDIT.

ParametroTipoDescripcion
elementIdstringID del elemento HTML.

Retorna: boolean — true si la fecha existe en el calendario.

no_file(elementId)

Verifica que un input file tenga un archivo seleccionado (usa readFile internamente).

ParametroTipoDescripcion
elementIdstringID del input file

Retorna: boolean — true si hay archivo seleccionado

format_name_file(elementId, regexpStr)

Valida el formato del nombre del archivo seleccionado (lee el fichero con readFile). Si no hay fichero o la expresion regular no es valida devuelve false.

ParametroTipoDescripcion
elementIdstringID del input file
regexpStrstringExpresion regular para el nombre

Retorna: boolean — true si el nombre cumple el formato

type_file(elementId, allowedTypes)

Valida que el tipo MIME del archivo este entre los permitidos.

ParametroTipoDescripcion
elementIdstringID del input file
allowedTypesArray<string>MIME types permitidos

Retorna: boolean — true si el tipo esta permitido

max_size_file(elementId, maxSize)

Valida que el tamano del archivo no exceda el maximo en bytes.

ParametroTipoDescripcion
elementIdstringID del input file
maxSizenumberTamano maximo en bytes

Retorna: boolean — true si el archivo no supera el tamano

personalized(elementId, entityInstance, attrName, allValues)

Ejecuta una validacion personalizada definida en la clase de la entidad. Invoca el metodo nombreAtributo_personalized_validation(value, allValues) sobre entityInstance.

ParametroTipoDescripcion
elementIdstringID del elemento HTML
entityInstanceobjectInstancia de la clase de la entidad
attrNamestringNombre del atributo
allValuesobjectTodos los valores del formulario

Retorna: boolean|string — true si valido, o codigo de error

ValidateFieldsForm ValidateFieldsForm_Class.js

Clase que crea los campos HTML individuales de cada atributo, los rellena con valores de prueba y ejecuta las validaciones.

createField(attrName, attrDef, action)

Crea un elemento HTML del formulario segun la definicion del atributo. Soporta los tags input, file, select, checkbox, radio y textarea. Si el atributo es de tipo file pero la accion no usa validaciones especificas de fichero (no_file, type_file, format_name_file, max_size_file), se crea como input de texto para poder rellenarlo con un nombre.

ParametroTipoDescripcion
attrNamestringNombre del atributo (usado como id y name del elemento creado).
attrDefobjectDefinicion del atributo (campos html, validations...).
actionstringAccion en curso (ADD, EDIT, SEARCH). Opcional; permite decidir si un campo file se trata como texto para esa accion.

Retorna: HTMLElement — El elemento HTML creado.

setValue(element, value, fileData)

Asigna un valor al elemento. Para campos file, crea un mock File.

ParametroTipoDescripcion
elementHTMLElementElemento HTML
value*Valor a asignar
fileDataobjectDatos del fichero {name, type, size}
executeValidation(validationType, elementId, param, entityInstance, attrName, allValues)

Ejecuta una validacion atomica sobre un campo utilizando Validations.

ParametroTipoDescripcion
validationTypestringTipo de validacion
elementIdstringID del elemento
param*Parametro de validacion
entityInstanceobjectInstancia de la clase entidad
attrNamestringNombre del atributo
allValuesobjectValores del formulario

Retorna: boolean|string — Resultado de la validacion

createTestContainer()

Crea un contenedor oculto en el DOM para campos de prueba.

Retorna: HTMLElement

cleanup()

Elimina el contenedor oculto de pruebas del DOM.

project project_Class.js

Clase de la entidad project. Existe porque end_date_project necesita una validacion cruzada con start_date_project: comprobar que la fecha de fin es posterior a la de inicio. La validez del formato y de la fecha en si (dias por mes, bisiestos) se resuelve con la validacion atomica date() de Validations; aqui solo se implementa lo que no puede ser atomico.

parseDate(str)

Metodo auxiliar usado internamente por end_date_project_personalized_validation. Convierte una cadena dd/mm/aaaa en un objeto Date si la fecha existe en el calendario, sin restriccion de rango de anos.

ParametroTipoDescripcion
strstringCadena con la fecha.

Retorna: Date|null — objeto Date o null si la cadena no es una fecha real.

end_date_project_personalized_validation(value, allValues)

Validacion personalizada para end_date_project. Comprueba que la fecha de fin sea posterior a la fecha de inicio. La validez de la fecha como tal (dias por mes, bisiestos) la garantiza la validacion atomica date(), que se ejecuta antes en la cadena de validaciones de la estructura.

ParametroTipoDescripcion
valuestringValor de la fecha de fin en formato dd/mm/aaaa.
allValuesobjectResto de valores del formulario; se espera start_date_project.

Retorna: boolean|string — true si la fecha de fin es posterior a la de inicio, o end_date_project_personalized_KO.

Clases para Comprobar el Submit

TestSubmit TestSubmit_Class.js

Clase que ejecuta las pruebas de submit del formulario. Verifica todos los atributos para cada accion (ADD, EDIT, SEARCH) sin realizar peticiones al backend. Si el fichero nombreentidad_TestSubmit.js no esta presente, la clase es capaz de generar las pruebas de submit automaticamente a partir de las pruebas de atributo (nombreentidad_pruebas): construye un "baseline" valido por accion y, para cada prueba, aplica los valores sobre ese baseline.

constructor(entityName)

Si window[entityName + "_TestSubmit"] no esta definido, llama a generateTestSubmitFromPruebas() y marca el flag autoGenerated = true (se indica visualmente en la ventana de resultados).

ParametroTipoDescripcion
entityNamestringNombre de la entidad a testear
generateTestSubmitFromPruebas()

Construye las pruebas de submit a partir del array nombreentidad_pruebas. El proceso es: por cada accion se acumula un registro baseline con los valores validos disponibles en las pruebas con resultado esperado true; despues, por cada prueba (sea de exito o error), se crea una entrada que aplica los valores de la prueba sobre el baseline de su accion. De esta forma una sola "ventana de salida" cubre tambien las pruebas de submit sin necesidad de escribir el fichero.

Retorna: Array — mismo formato que nombreentidad_TestSubmit.

run()

Ejecuta todas las pruebas de submit y muestra el modal de resultados.

Retorna: void

renderSummary()

Pinta en el cuerpo del modal la tabla resumen por accion (ADD, EDIT, SEARCH) y enlaza el boton "Ver detalle".

Retorna: void

renderDetails()

Pinta en el cuerpo del modal el detalle prueba a prueba con resultado esperado / obtenido y un boton "Volver al Resumen".

Retorna: void

CheckSubmit(accion, atributosValores)

Comprueba todos los atributos requeridos para la accion. Acumula todos los errores encontrados y los devuelve, o true si no hay ninguno. No realiza llamada al backend.

ParametroTipoDescripcion
accionstringAccion (ADD, EDIT, SEARCH)
atributosValoresobjectObjeto {nombreAtributo: valor, ...}

Retorna: boolean|Array<string> — true si todo correcto, o un array con todos los codigos de error encontrados.

runAllSubmitTests()

Ejecuta todas las pruebas definidas en nombreentidad_TestSubmit.

Retorna: object — {byAction, total, totalCorrect, totalIncorrect, allDetails}

formatValueForCell(value)

Igual que el de TestForm: convierte valores a representacion compacta para la tabla de detalle. Las cadenas de caracter repetido se muestran como "X".repeat(N).

ParametroTipoDescripcion
value*Valor a formatear.

Retorna: string

buildSummaryHTML(results)

Genera el HTML del bloque de resumen por accion con los contadores de correctas/incorrectas y el boton de detalle.

ParametroTipoDescripcion
resultsobjectResultado de runAllSubmitTests().

Retorna: string — HTML.

buildDetailsHTML(results)

Genera el HTML de la tabla con el detalle de cada prueba de submit: accion, descripcion, atributos/valores, resultado esperado, resultado obtenido e indicador CORRECTO/INCORRECTO.

ParametroTipoDescripcion
resultsobjectResultado de runAllSubmitTests().

Retorna: string — HTML.

exportJSON()

Descarga un fichero tests_submit_<entidad>.json con el resultado completo de las pruebas de submit: resumen por accion (total, correctas, incorrectas) y detalle prueba a prueba (accion, descripcion, resultado esperado, resultado obtenido, correcta). Si las pruebas fueron auto-generadas se indica el campo autoGeneradas: true.

Retorna: void
